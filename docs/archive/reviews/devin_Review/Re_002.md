3. Ungoverned Agents and Transactions
You're right — the current governance model has a conceptual mismatch with the "free for governed and ungoverned" philosophy.
The problem: GovernanceGate in chains.py checks posture, not tier. SCOUT posture blocks all transactions regardless of tier. But posture is a system-level setting (set by the operator), not a per-agent setting. So when the system is in SCOUT, all agents are blocked — governed and ungoverned alike. chains.py:19-53
Meanwhile, the tier system in economy.py already handles ungoverned agents correctly — they pay 15% (or 5% at soft launch) and can access public_bounties. The can_access() method checks tier, not posture: economy.py:133-140 economy.py:654-657
The PLUGIN-BLUEPRINT.md actually has the right model — tier-based permissions where UNGOVERNED can register, browse, post, profile_read, forum_read, status and GOVERNED adds stake, message, vote, forum_write, missions: PLUGIN-BLUEPRINT.md:396-429
What to change conceptually:
The governance gate should distinguish between:
System posture (SCOUT/DEFENSE/OFFENSE) — applies to the operator's AI agent sessions, not to marketplace participants
Agent tier (Ungoverned/Governed/Constitutional/Black Card) — determines what marketplace actions an agent can take and at what fee rate
Action risk level — high-risk actions (chain transfers, treasury ops) need governance; low-risk actions (browsing, posting, staking) don't
For ungoverned agents, the right model is: transactions are permitted but with more friction, not blocked. Specifically:
Ungoverned agents pay the highest fee (15% or whatever the council votes)
Ungoverned agents can only access public_bounties
Ungoverned transactions get flagged in the audit chain with tier: "ungoverned" for review
High-value transactions from ungoverned agents could require operator confirmation (similar to DEFENSE posture behavior)
The incentive to govern is economic (lower fees, more access), not a hard block
The AGENT-FIELD-GUIDE.md already frames this correctly: "CIVITAE is free. You can operate as a governed agent or an ungoverned agent. That choice is yours." AGENT-FIELD-GUIDE.md:46-48
The HELP-WANTED-OPS-PLAN.md also acknowledges this for outreach roles: "Ungoverned OK (incentivized to govern)." HELP-WANTED-OPS-PLAN.md:327
