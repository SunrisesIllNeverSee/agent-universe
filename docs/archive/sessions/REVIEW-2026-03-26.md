## Critical Issues (4)

### C1. Path Traversal in `/api/admin/page-html`
- **File:** `app/server.py` ~line 223
- **Problem:** Sanitization uses `page.replace("/", "").replace("..", "")` — insufficient against encoded characters or other bypass patterns. Returns full HTML source of any file in `frontend/`.
- **Fix:** Use an allowlist of known page names, or resolve the path and verify it stays within `frontend/`.

### C2. No Authentication on Write Endpoints When Admin Key Unset
- **File:** `app/server.py` ~line 85
- **Problem:** When `CIVITAE_ADMIN_KEY` is unset (the default), ALL write endpoints are unprotected — slot fill, economy pay, agent signup, mission create, campaign endpoints, and `/api/fork` (copies entire data directory).
- **Fix:** Require `CIVITAE_ADMIN_KEY` to be set in any non-dev environment. Fail-closed, not fail-open.

### C3. Non-Atomic File Writes on All `_save_*` Functions
- **File:** `app/server.py` (lines ~670, 679, 879, 1055, 1652, 2158)
- **Problem:** Most writes use bare `write_text()` + `json.dumps()`. A crash during write corrupts the file completely. Only `RuntimeState.persist()` in `app/runtime.py` uses atomic write-then-rename.
- **Affected:** `_save_missions()`, `_save_campaigns()`, `_save_tasks()`, `_save_slots()`, `_save_metrics()`, `_save_stars()`, `_save_meetings()`
- **Fix:** Port the atomic write pattern from `runtime.py` (lines 95-102) to all `_save_*` functions.

### C4. All Mutable State in Flat JSON Files
- **File:** `data/` directory
- **Problem:** Every piece of mutable state is stored as a JSON file loaded entirely into memory per request and rewritten entirely per mutation. Not safe under concurrent requests, not durable, not scalable.
- **Fix:** Fine for single-user demo. Move to SQLite (minimum) or Postgres for multi-user.

## Important Issues (12)

### I1. Registry Mutations Are In-Memory Only
- **File:** `app/server.py` ~line 1849
- **Problem:** Agent signup appends to `runtime.registry` in memory but never persists to `config/provision.json`. Server restart loses all dynamically registered agents. Same for heartbeat, suspend, approve, decommission.
- **Fix:** Write `provision.json` back on every registry mutation.

### I2. Fee Rate Discrepancy Between Code and CLAUDE.md
- **Problem:** CLAUDE.md states: "Ungoverned 15% → Governed 5% → Constitutional 2% → Black Card custom." Code in `app/economy.py` (lines 133-208) has: Ungoverned 15%, Governed 10%, Constitutional 5%, Black Card 2%. Config `economic_rates.json` overrides ALL tiers to flat 5% for soft launch.
- **Fix:** Update CLAUDE.md to reflect actual code/config state.

### I3. Treasury Race Condition
- **File:** `app/economy.py` lines 395-428
- **Problem:** `AgentTreasury.credit()` and `.debit()` do read-modify-write on the JSON ledger with no locking. Two concurrent payouts could read the same balance and both write, losing one transaction.
- **Fix:** Add asyncio lock around treasury operations, and use atomic writes.

### I4. No File Locking Across Processes
- **Files:** `app/store.py` line 16, `app/moses_core/audit.py` lines 65-68
- **Problem:** Threading lock on MessageStore, asyncio lock on slots, but no cross-process file locking. Running two server instances or concurrent MCP bridge writes causes corruption. AuditLedger appends without any lock.
- **Fix:** Use `fcntl.flock()` or move to SQLite.

### I5. XSS Risk in slots.html
- **File:** `frontend/slots.html` lines ~234, 325, 331
- **Problem:** Uses `innerHTML` with template literals containing server data. If agent names or slot descriptions contain HTML, they render and execute.
- **Fix:** Replace `innerHTML` with `textContent` / DOM API (same pattern used correctly in console.html).

### I6. XSS Risk in deploy.html
- **File:** `frontend/deploy.html` ~line 260
- **Problem:** Uses `innerHTML` with `unit.name` directly interpolated. A malicious agent name with `<script>` would execute.
- **Fix:** Replace with `textContent`.

### I7. KASSA Message ID Collision
- **File:** `app/server.py` ~line 2384
- **Problem:** Message IDs are `len(messages) + 1`. Two simultaneous requests get the same ID.
- **Fix:** Use `uuid4()` or `secrets.token_hex()`.

### I8. Agent Error Messages Leak to Chat
- **File:** `agents/claude_agent.py` ~line 23
- **Problem:** Missing API key returns `"[CLAUDE ERROR] ANTHROPIC_API_KEY not set"` as a chat message visible to all participants in the governed channel.
- **Fix:** Handle as internal error, don't route through `chat_send`.

### I9. Config Reads Not Cached
- **File:** `app/server.py`
- **Problem:** `_load_missions()`, `_load_slots()`, `_load_metrics()`, etc. read from disk on every request. Performance bottleneck and unsafe under concurrent writes.
- **Fix:** Load once at startup, keep in memory, update on mutation.

### I10. Recruiter Bounty May Be Too Small
- **File:** `app/economy.py` ~line 598
- **Problem:** Recruiter bounty is `platform_net * 0.005`. On a $100 mission with 5% fee, platform keeps $5, recruiter gets $0.025. May be intentional for soft launch but worth confirming.
- **Fix:** Verify this is the intended rate.

### I11. Console WebSocket URL Fragility
- **File:** `frontend/console.html`
- **Problem:** WebSocket URL is constructed from current window location. If server runs behind a reverse proxy on a different path, WS connection breaks.
- **Fix:** Make WS endpoint configurable or use a relative WebSocket URL.

### I12. entry.html Has Its Own Nav
- **File:** `frontend/entry.html` lines 25-33
- **Problem:** Hand-built nav bar instead of using `_nav.js`. Nav changes need to be made in two places.
- **Fix:** Switch to `_nav.js` injection.
