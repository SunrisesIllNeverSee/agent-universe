## CRITICAL DISTINCTIONS

These distinctions matter for positioning, communication, sales conversations, and product development decisions. When someone misunderstands DEPLOY, it usually falls into one of these categories.

---

## IT IS NOT A GAME

DEPLOY uses game logic, game metaphors, and game interface patterns. It is not a game.

The output is not a score. The output is a real marketing campaign, a real codebase, a real financial operation, a real inbox at zero. The stakes are real. The results are real. The ROI is real.

The game logic is the *interface layer* — the intuitive language that eliminates the cognitive overhead of agentic configuration. The work underneath is completely serious.

---

## IT IS NOT A SIMULATION

DEPLOY includes simulation as one objective type — Scout posture, recon campaigns, testing before committing. That is one mode among many.

DEPLOY itself is not a simulation platform. It is an operations platform that includes simulation capability. The distinction matters: you don't use DEPLOY to practice. You use DEPLOY to operate. Simulation is one tool in that operation, not the product's identity.

---

## IT IS NOT COMMAND

COMMAND is the operational cockpit. Governance, systems management, agent definition, session control. Better chatting with AI at the highest level.

DEPLOY is the campaign engine. Objectives, formations, rosters, wins.

COMMAND makes you operate better. DEPLOY makes you win.

They work best together. They are not the same product. A COMMAND license does not include DEPLOY. A DEPLOY license does not include COMMAND.

---

## IT IS NOT THE DEPLOY FEATURE INSIDE COMMAND

COMMAND contains a deploy feature — a lightweight mission preset launcher. Three tabs, quick config, launch. Minimal friction, minimal depth.

DEPLOY · by MO§ES™ is the full expression of that concept. Full formation library, deep campaign configuration, roster management, squad saves, campaign history, live dashboards, rotation triggers, per-slot posture overrides, cross-domain objective support.

The deploy feature inside COMMAND is a taste. DEPLOY is the full product.

---

## IT IS NOT AN AGENT BUILDER

DEPLOY does not build agents. Agents are built in COMMAND or brought in from external sources. DEPLOY orchestrates agents that already exist.

You bring your players. DEPLOY is the field and the playbook.

---

## IT IS NOT A WORKFLOW AUTOMATION TOOL

Tools like Zapier, Make, n8n, UiPath automate predefined workflows. They execute known processes reliably.

DEPLOY orchestrates autonomous agents toward objectives. The path to the objective is not predefined — agents reason, decide, and adapt within the formation and posture constraints you set. This is not automation of known processes. This is deployment of autonomous capability toward defined goals.

---

## IT IS NOT AN ENTERPRISE ORCHESTRATION PLATFORM

Tools like Camunda, Adobe Agent Orchestrator, and Microsoft Copilot Studio are built for enterprise IT departments. Visual flowcharts, BPMN diagrams, integration schemas, compliance frameworks.

DEPLOY is built for operators — founders, traders, growth teams, researchers, builders — who think in outcomes, not processes. The interface language is formations and campaigns, not nodes and workflows.

Enterprise buyers can and will use DEPLOY. But the product is not designed around enterprise procurement cycles, IT department requirements, or organizational compliance structures.

---

## IT IS NOT A SOCIAL OR MARKETPLACE PLATFORM

DEPLOY does not include agent discovery, agent marketplaces, or social features in V1.

Squad sharing, formation marketplaces, and collaborative campaign features are future expansion considerations. They are not V1 scope.

---

## IT IS NOT MO§ES™ CORE

DEPLOY is a frontend product sitting on top of the MO§ES™ architecture. It does not include:

- MO§ES™ core runtime
- Compression engine
- Mediator internals
- Invariant enforcement engine
- Semantic compression internals
- Lineage systems
- Patent or trademark rights

White-label buyers receive the recipe. Not the kitchen. The kitchen stays with Ello Cello LLC permanently under all agreements.

---

## IT IS NOT FREE

Command Minis is free. DEPLOY is a commercial product with wave cascade pricing. The free tier entry point to the ecosystem is Minis, not DEPLOY.
