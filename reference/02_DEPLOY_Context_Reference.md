## WHERE THIS CAME FROM

DEPLOY did not begin as a product. It began as a button.

The ask was simple: add a deploy feature to COMMAND. What happened in the process of designing that feature revealed a system deep enough to warrant its own product. The deploy feature inside COMMAND became the lite version. DEPLOY became the full expression.

The progression that revealed the product:

**Step 1** — Deploy needs to mean something. It's not just launching agents. It's strategic orchestration.

**Step 2** — Formations. Sports. Military. Gaming. Business. People already know how to think this way. The metaphor is the instruction.

**Step 3** — Posture. Scout / Defense / Offense. Every objective, every domain, every formation runs through three states. That's the compression.

**Step 4** — The Sims insight. Same "set it and watch it run" logic. Same roster building, formation setting, objective defining. But the output is real work — not a simulation score.

**Step 5** — The split. The depth of the system exceeded what belongs inside COMMAND. Two products emerged naturally from one conversation.

## REFERENCE: THE SIMS / ROLLERCOASTER TYCOON PARALLEL

The most useful reference frame for understanding DEPLOY is not enterprise software. It is simulation games.

In The Sims, RollerCoaster Tycoon, Football Manager — you set the objectives, you build the environment, you design the roster, you set the rules — and then you let it run. You are not playing the game. You are the architect of the game. The agents are the inhabitants.

The critical distinction from those games:

**Those games simulate life. DEPLOY operates life.**

Same interface logic. Same setup-and-watch satisfaction. Same roster building and formation design. But the output is a real marketing campaign, a real codebase, a real financial operation — not a simulation score.

The learning curve is already solved. Billions of people already know how to think in systems. DEPLOY points that intuition at real work.

## REFERENCE: SPORTS ANALOGY DEPTH

The sports metaphor is not superficial. It carries real functional meaning at every layer.

**The field** — every sport has a surface with zones. Formations map to that surface spatially.

**The lineup** — you don't design plays without knowing your roster. Players before plays.

**The playbook** — pre-designed formations for known situations. Coaches don't invent on the fly.

**Substitutions** — mid-game agent swaps when conditions change.

**The referee** — governance. Rules enforced throughout, not just at the start.

**The scoreboard** — campaign dashboard. Real-time progress against win conditions.

**Practice vs game day** — simulation objective (Scout posture) vs live execution (Offense posture).

**The bench** — agents on standby, available when needed.

**The opponent** — the objective, the market, the problem. Always present even when unnamed.
