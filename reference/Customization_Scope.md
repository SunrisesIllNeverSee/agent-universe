## PRINCIPLE

Every seat includes UI customization rights. The scope of included customization is defined, bounded, and signed off on as part of the seat agreement. The full component, label, and function list is not public — it is disclosed to the licensee at transfer and forms part of the signed agreement.

What is visible publicly: the scope boundary. What is customizable vs. what is upcharged. Not the list itself.

---

## INCLUDED WITH SEAT

Customization included in all seat tiers at no additional cost:

- **Color system** — full theme swap via CSS variable replacement. Every color in the UI is controlled through a centralized token system. One swap covers the entire product.
- **Typography** — font family and weight swaps via the same token system
- **Labels** — button text, section headers, UI copy, nomenclature
- **Brand application** — logo, wordmark, product name as displayed in the interface

These changes require no structural code modification. They are surface-level and fully within scope of the license.

---

## UPCHARGED — AVAILABLE AT ADDITIONAL COST

Changes outside included scope, available by separate agreement:

- Layout modifications — repositioning, restructuring, or reorganizing UI components
- Functional additions — new features, new views, new workflows not present in the base product
- Component-level changes — modifying how existing components behave, not just how they look
- Integration work — connecting the product to external systems, APIs, or data sources
- Any change requiring structural code modification

Upcharge rate and scope defined per engagement. Quoted separately from the seat license.

---

## THE FULL LIST — TRANSFER ONLY

The complete definition of every customizable element — buttons, labels, functions, components, interactions — is internal IP.

It is:
- Not published publicly
- Not included in marketing materials
- Not shared prior to agreement execution

It is disclosed to the licensee as part of the seat transfer process and signed off on as a defined schedule within the agreement. The licensee receives full clarity on exactly what is and is not customizable under their license class at the point of transfer — not before.

This protects the architecture. A public component and function list is a product map. It stays behind the agreement.
